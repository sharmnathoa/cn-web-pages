PG模拟器：轻量查询与对照平台

在编写、调试或回溯 PostgreSQL 相关逻辑时，一个可交互的模拟环境能大幅减少上下文切换。PG模拟器围绕查询验证与结果对照设计，支持快速输入 SQL 片段、观察执行计划梗概，以及对不同写法进行并列比对。

适合需要频繁验证函数行为、索引策略或窗口语义的开发与 DBA 人员，也适合作为学习阶段的辅助沙箱。

即时查询

  • 无需本地搭建实例，浏览器内模拟常见 PG 语法
  • 支持 SELECT、JOIN、聚合、子查询等核心结构
  • 返回模拟结果集与执行步数概览
  • 适合验证逻辑正确性，或快速重现文档中的示例

并列对照

  • 将两种写法分别输入左右面板，同时观察输出差异
  • 对比索引提示、CTE 展开、DISTINCT 与 GROUP BY 等效性
  • 高亮行数差异与结果行变化，辅助推理

延伸阅读

  • 每条查询之后附带相关 PG 文档片段链接
  • 根据关键词自动匹配官方手册条目
  • 可折叠的上下文解释，减少外部搜索

▎使用建议与边界

起点 —— 将 PG模拟器 作为快速验证台：编写一条带窗口函数的查询，对比 ROWS/RANGE 区别；或者测试不同类型转换的行为。模拟器返回的是近似结果，适用于逻辑预检，不适合作为生产级性能推断依据。

对照方法 —— 在左侧面板输入原始查询，右侧面板输入改写版本，观察结果集是否一致。尤其适合检查等价改写（如 EXISTS 与 IN，UNION 与 UNION ALL 的语义差异）。

延伸阅读场景 —— 当你对某个语法点不确定时，在模拟器中输入并执行，页面底部的延伸卡片会推荐相关 PG 手册段落。例如输入 LATERAL 子查询，延伸区将展示官方说明摘要。

⚡ 适合日常 SQL 写作、教程跟随、对比实验；不适合存储过程调试或精确性能测量。

常见疑问 · 摘要

Q: 模拟的 SQL 范围有多广？—— 覆盖 DML 主要子集（SELECT/JOIN/WHERE/GROUP BY/HAVING/WINDOW/CTE），不包含 DDL 持久化及 PL/pgSQL 完整语法。
Q: 结果与真实 PG 实例一致吗？—— 核心逻辑基于 PG 语义实现，数据类型、聚合、窗口函数行为经对照测试，但复杂触发器、自定义类型、扩展模块不在模拟范围内。
Q: 对照功能支持多少条语句？—— 同时支持两组输入（各不超过 500 字符），对比结果行及异常提示，适合短查询比对。

访问说明

PG模拟器运行于静态前端环境，查询在浏览器本地完成模拟，无数据外传。无需注册，不收集个人信息。服务由独立团队维护，与 PostgreSQL 官方无直接关联。如果你需要与生产数据库完全一致的行为验证，建议同时在本地 PG 实例中复测。

使用前建议阅读站点内的 已知限制 页面，了解当前版本未覆盖的功能边界。
